# Purpose

This document defines the core business entities of LVN Atlas.

It intentionally avoids implementation details such as databases, APIs, or frameworks.

Instead, it answers one question:

**What concepts exist in the business domain?**

---

# Core Domain

Relocation Decision Planning

The platform exists to help users evaluate whether relocating to another city is financially sustainable.

---

# Aggregate Root

## User

The User is the primary owner within the system.

A User can:

- maintain one profile
- create multiple Migration Blueprints
- save relocation scenarios
- contribute anonymous Migration Journeys

---

# Entities

## User Profile

Represents personal information used to personalize recommendations.

Contains:

- Occupation
- Current City
- Family Information
- Lifestyle
- Housing Preference
- Transportation Preference
- Dietary Preference

Relationship

User (1) → Profile (1)

---

## Migration Blueprint

Represents one relocation scenario.

Examples:

- Jaipur → Mumbai
- Jaipur → Pune

Contains:

- Origin City
- Destination City
- Expected Salary
- Planned Relocation Date
- Budget
- Atlas Score

Relationship

User (1) → Migration Blueprint (N)

---

## Budget

Represents projected monthly finances.

Contains:

- Income
- Expenses
- Savings
- Disposable Income

Relationship

Migration Blueprint (1)
↓

Budget (1)

---

## Expense Category

Shared list of expense types.

Examples:

- Rent
- Groceries
- Utilities
- Healthcare
- Transportation
- Entertainment

Relationship

Expense Category (1)

↓

Expense Items (N)

---

## Expense Item

Represents one expense inside a budget.

Examples

Rent

₹18,000

Groceries

₹6,200

Internet

₹900

Relationship

Budget (1)

↓

Expense Item (N)

---

## City

Represents a supported city.

Contains:

- Name
- State
- Country

Future:

- Cost of Living
- Population
- Community Statistics

---

## Migration Journey

Represents an anonymous real relocation shared by a user.

Contains:

- Origin City
- Destination City
- Occupation
- Salary Range
- Expense Breakdown

Relationship

User (1)

↓

Migration Journey (N)

---

## Atlas Score

Represents relocation readiness.

Generated from:

- Income
- Expected Savings
- Housing Cost Ratio
- Family Size
- Community Data

Atlas Score is **calculated**, not manually entered.

---

# Business Rules

A User must have exactly one Profile.

A User may create many Migration Blueprints.

A Blueprint must reference exactly one Origin City.

A Blueprint must reference exactly one Destination City.

A Blueprint owns exactly one Budget.

A Budget contains multiple Expense Items.

Expense Categories are shared globally.

Migration Journeys are anonymous.

Atlas Score is generated by the system.
